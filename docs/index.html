No LLM at runtime. Each robot carries the reasoning onboard. LMM decomposes the task offline, an RNN encodes the logic, a GNN executes it. 100% accuracy across 7 tasks, real RoboMasters hardware. http://ieeexplore.ieee.org/document/11601083

The clever bit: finite automata and RNNs are structurally compatible. Supervised training aligns hidden states to automaton states explicitly. So you get LLM-level task decomposition in a model small enough to run onboard each robot. Open-access version: https://arxiv.org/abs/2509.24575
The honest caveat: aggregating neighbor observations into atomic propositions is still hand-engineered per task. Authors flag this themselves. Until that's automated from language, you need a human in the loop for each new task type. How big a blocker is that for your use case?
